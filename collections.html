Collections

In collaboration with Caitlyn Hampton, I designed Collections, a tool to create and exhibit series of stories.


The Problem

Current story collections on The Atlantic (i.e. channels, catagories, special projects) are stale, inflexible layouts that do little more than list articles.
We want to package articles together in beautiful, distinctive, and useful ways for readers while reducing custom builds.


Research

  • Stakeholder interviews to uncover needs and wishes for editorial, design, sales, data, and product teams
  • Competitive research of other media sites' collections pages
    • How do others organize stories to provide delightful and immersive experiences to readers?

Defining Goals

  • Focusing on user outcomes, we identified 5 user archetypes: readers, editorial/creative, sales/clients, product, growth.
  • For each user, an empathy map was created that highlighted what they said, thought, felt, and did as part of their role:

Empathy map for editorial/creative user

  • Next, many user stories were made, each defining a successful scenario.
    • User story formula: who (is the user), what (is the need to be met), how (to measure success).
    • ex) A reader found a special project on Twitter, read at least 2 stories, and then tweeted the special project himself.

Organizing user stories

  • Main goals:
    • Empower editors with the flexibility to create story collection layouts that highlight their editorial coverage while impressing readers
    • Give readers collection pages that don't just host articles, but serve as rich story-telling pages themselves
    • Reduce # of unique page builds by the product team, saving time & resources

Proposed Solution

  • Choose a scenario-driven template:

* Conceptual scenarios

  • Each template has predetermined zones: header, curated, river (list) of articles. Different variations of each zone can be selected to display the component differently:
  • This solution provides structure and flexibility, allowing editors to create uniquely-structured pages while maintaining brand cohesion. New variants can also be created and added to the tool.

  • Paper prototyping: we took our designs and paper prototyped with editors to see how they would build pages:

  • User testing: editorial wanted text components, so using Usertesting.com, we set out to see if text provided value to these pages:

Left side: Subheaders, light text. Right side: Only subheaders

    • Results showed that text and subheaders provided important hierarchy and outline to a collection of stories, while no text left readers with a weaker impression of the collection's mission.
